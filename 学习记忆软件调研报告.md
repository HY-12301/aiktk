# 二、核心算法详解

## 2.1 SM-2 算法（SuperMemo / Anki）

**SM-2** 由波兰科学家 Piotr Wozniak 于 1987 年提出，是间隔重复领域最具影响力的算法。现代闪卡软件（Anki、RemNote、Mnemosyne）大多基于此。

### 核心数据结构

每张闪卡维护三个关键参数：

| 参数 | 含义 | 初始值 |
|------|------|--------|
| `interval` | 当前复习间隔（天） | 1 |
| `repetitions` | 成功复习次数 | 0 |
| `EF`（Easiness Factor） | 易度因子 | 2.5 |

### 核心公式

```
新间隔 = 当前间隔 × EF

EF 更新公式（用户评分为 q，0-5 分）：
若 q ≥ 3：
  EF' = EF + (0.1 - (5-q) × (0.08 + (5-q) × 0.02))
  下限约束：EF ≥ 1.3
若 q < 3：
  间隔重置为 1 天，repetitions 归零
```

### 评分标准

| 分数 | 含义 | 行为 |
|------|------|------|
| 0 | 完全遗忘 | 重置间隔，EF - 20% |
| 1-2 | 答错但记住一点 | 重置间隔，部分降低 EF |
| 3 | 努力回忆起来 | 间隔 × EF，EF 不变 |
| 4 | 正确但犹豫 | 间隔 × EF，EF + 10% |
| 5 | 轻松答对 | 间隔 × EF × 1.3，EF + 15% |

### 算法逻辑

```
function sm2(quality, card):
    if quality >= 3:
        if card.repetitions == 0:
            card.interval = 1
        elif card.repetitions == 1:
            card.interval = 6
        else:
            card.interval = round(card.interval * card.EF)
        card.repetitions += 1
    else:
        card.repetitions = 0
        card.interval = 1

    card.EF = card.EF + (0.1 - (5-quality) * (0.08 + (5-quality)*0.02))
    if card.EF < 1.3:
        card.EF = 1.3
    card.nextReview = today + card.interval
```

### Anki 对 SM-2 的改进

1. **学习阶段**：新增卡片先通过固定步骤 `30m → 2h → 2d`，然后进入指数阶段
2. **延迟奖励**：如果卡片在最佳日期之后才复习且仍然答对，间隔奖励 `延迟天数 / 难度分隔值`
3. **随机噪声**：间隔添加微小随机扰动，防止同一天创建的卡片总是撞在一起
4. **最少间隔下限**：EF 不能低于 130%（1.3），以防止无限频繁复习

## 2.2 莱特纳盒子系统（Leitner Box）

**Leitner Box** 是一个物理/数字版本的初级间隔重复系统，是 SM-2 的前身。

### 核心逻辑

```
5 个盒子，复习间隔依次为：
Box 1: 每 1 天
Box 2: 每 3 天
Box 3: 每 7 天
Box 4: 每 14 天
Box 5: 每 30 天

规则：
- 所有新卡片放入 Box 1
- 答对：卡片升到下一个盒子
- 答错：卡片回到 Box 1（不管当前在哪个盒子）
```

### 优势与局限

| 优势 | 局限 |
|------|------|
| 实现极简，无需计算 | 间隔固定，不个性化 |
| 物理版本可行 | 答错一次全盘重置，过于严厉 |
| 直观易懂 | 不对用户遗忘特性建模 |

## 2.3 Half-Life Regression（Duolingo）

Duolingo 在 2016 年 ACL 论文中提出 **半衰期回归模型（HLR）**，是间隔重复领域的重大进步。

### 核心概念

**半衰期**：学习者有 50% 概率忘记该知识点的时间点。算法的核心任务是精确预测这个时间点。

### 数学本质

HLR 不是固定公式，而是一个**机器学习回归模型**：
- 输入：多维度特征（用户历史、单词难度、间隔时间、练习结果等）
- 输出：该单词对该用户的半衰期（天）
- 训练数据：Duolingo 平台上的 **1300 万条学习轨迹**
- 效果：记忆回忆预测误差降低 **45%** 以上

### 相对于 SM-2 的关键进步

| 维度 | SM-2 | HLR |
|------|------|-----|
| 建模对象 | 一维 EF 因子 | 多维特征向量 |
| 个性化 | 仅通过 q 值反馈 | 每个用户、每个单词独立建模 |
| 可训练性 | 公式固定 | 从大规模数据中学习 |
| 精确度 | 经验公式 | ML 回归模型 |

---

## 2.4 BirdBrain AI 引擎（Duolingo）

Duolingo 在 HLR 基础上演进为 **BirdBrain**，统一了"何时复习"和"复习什么"两个核心决策。

### BirdBrain V1（双参数模型）

类似国际象棋的 **Elo 评分系统**：
- 每位用户维护**能力分数**
- 每道练习维护**难度分数**
- 答对：升高用户能力、降低练习难度
- 答错：降低用户能力、升高练习难度
- 变动幅度取决于结果是否"出乎意料"

### BirdBrain V2（40 维学习者画像）

- 引入 **LSTM 循环神经网络**
- 将每位学习者的完整互动历史压缩为 **40 个数字** 的向量
- 每次回答后基于先前状态 + 当前练习结果更新向量
- 能捕获**细粒度的能力差异**（如用户擅长过去时却挣扎于将来时）
- 每天处理 **10 亿次练习**，**14 毫秒**内决定下一道练习

### 端到端统一

在最新架构中：
- 间隔记忆预测（何时复习）和练习筛选（复习什么）
- 由同一神经网络同时输出
- 不再需要独立的两套系统同步

# 三、各平台详细分析

## 3.1 SuperMemo

| 维度 | 描述 |
|------|------|
| **算法演进** | SM-2（1987）→ SM-5 → SM-8 → SM-11 → SM-15 → SM-17 → SM-18 |
| **核心理念** | 以最小复习次数达成最高长期保持率（实测 6 个月保持率 > 85%） |
| **最新算法** | SM-17/SM-18 引入双变量模型：「稳定性」（stability）+「可提取性」（retrievability） |
| **内容组织** | 将知识分解为最小原子单元（item），支持增量阅读 |
| **掌握度** | 通过评分（0-5）实时反馈，EF 因子动态调整 |
| **个性化** | 核心算法封闭（商业机密），仅开放 SM-2 |

### SM 版本演进关键节点

| 版本 | 年份 | 核心改进 |
|------|------|----------|
| SM-2 | 1987 | 引入 EF 因子，间隔指数增长 |
| SM-5 | 1989 | 优化间隔计算公式，考虑历史遗忘数据 |
| SM-8 | 1995 | 引入遗忘矩阵，更精确的遗忘预测 |
| SM-11 | 1999 | 增量阅读功能（interleaved reading + review） |
| SM-17 | 2016 | 双变量模型：稳定性 + 可提取性 |
| SM-18 | 2019 | 进一步优化稳定性预测的精确度 |

## 3.2 Anki

| 维度 | 描述 |
|------|------|
| **算法** | 改进版 SM-2 |
| **学习阶段** | 学习阶段（固定步骤）+ 指数阶段 + 重新学习阶段，三阶段管理 |
| **掌握度** | 4 级评分（Again / Hard / Good / Easy），映射到 SM-2 的 0-5 分 |
| **内容组织** | 牌组（Deck）→ 卡片（Card），支持自定义字段、CSS 模板、插件扩展 |
| **个性化** | 每张卡片独立的 EF 因子，支持插件扩展算法 |
| **优势** | 开源、社区庞大、跨平台同步、高度可定制 |
| **局限** | 算法相对简单（仅 SM-2）、对大规模知识图谱组织支持弱 |

### 三阶段管理模式

```
[新卡片] → 学习阶段（30m → 2h → 2d）
    ↓ 轻松
    指数阶段（间隔 = 上次间隔 × EF）
    ↓ 忘记
    重新学习阶段（同学习阶段，但恢复间隔 = 上次指数间隔 × 0.1）
```

## 3.3 Quizlet

| 维度 | 描述 |
|------|------|
| **算法** | 自适应间隔重复 + 掌握度追踪 |
| **学习模式** | Learn 模式：自动混合选择题、填空题、判断题，直到连续两次答对所有卡片 |
| **掌握度** | 二值 + 连续评分：卡片分为「已掌握」/「未掌握」，同时追踪连对次数 |
| **内容组织** | 学习集（Set）→ 术语卡片 + 自动生成练习形式 |
| **个性化** | AI 辅导 Q-Chat（基于 ChatGPT），自动解释错题、生成个性化复习计划 |
| **优势** | 7 种学习模式、游戏化（Match/Gravity）、丰富的现成学习集 |
| **局限** | 间隔重复算法不如 Anki 精确，更偏向短期应试 |

### Quizlet 掌握度模型

```
Learn 模式核心规则：
- 每张卡片经历多轮复习
- 连续答对 2 次 → 标记"已掌握"
- 答错 → 重新计数
- 所有卡片掌握 → 一轮完成
- 完成后再次混合复习已掌握卡片以强化长期记忆
```

## 3.4 Duolingo

| 维度 | 描述 |
|------|------|
| **算法** | Half-Life Regression + BirdBrain V2（LSTM 40 维画像） |
| **记忆调度** | 每单词独立预测半衰期，在遗忘临界点前安排复习 |
| **掌握度** | 连续概率值（遗忘概率 0-100%），非离散级别 |
| **内容组织** | 技能树 → 单元（Unit）→ 课程（Lesson）→ 练习（Exercise），逐级解锁 |
| **个性化** | 40 维 LSTM 画像 + GPT-4 角色扮演（Roleplay） |
| **优势** | 14ms 决策延迟、每日 10 亿次练习、45% 预测误差降低 |
| **局限** | 主要针对语言学习、算法不对外开放 |

### 个性化学习引擎架构

```
用户交互 → BirdBrain V2（LSTM）
                ├── 维度1: 词汇掌握
                ├── 维度2: 语法能力
                ├── ...
                └── 维度40: 其他特征
                    ↓
         统一输出：
         ├── 下一道练习选择（内容）
         ├── 记忆间隔建议（时间）
         └── 能力评估（掌握度）
```

## 3.5 Khan Academy（可汗学院）

| 维度 | 描述 |
|------|------|
| **算法** | 掌握学习法（Mastery Learning），非间隔重复的纯时间调度 |
| **核心理念** | 学生必须掌握上一层级内容后，才能进入下一层级 |
| **掌握度** | 0-100% 掌握度分数，基于练习正确率 + 持续性 |
| **内容组织** | 知识图谱（Knowledge Map）：层级化、有前置依赖的知识节点网络 |
| **个性化** | Khanmigo AI 导师（GPT-4），苏格拉底式引导，不自接给答案 |
| **优势** | 知识图谱可视化、完全免费、数学/科学领域深度覆盖 |
| **局限** | 不专门优化记忆保持，侧重理解而非背诵 |

### 知识图谱结构

```
数学
├── 算术
│   ├── 加减法
│   ├── 乘除法
│   └── 分数
├── 代数
│   ├── 变量与表达式
│   └── 方程（依赖：算术全部）
├── 几何（依赖：代数基础）
└── 微积分（依赖：代数+几何）
```

每个节点包含：视频教程 → 练习 → 测验 → 掌握度追踪

# 五、可借鉴的设计模式

## 5.1 算法层面

### 1. SM-2 作为基线模型
- **适用场景**：任何需要记忆调度的学习系统
- **实现要点**：EF 因子 + 3 阶段管理 + 评分反馈
- **可借鉴度**：★★★★★（最低成本实现有效间隔重复）

### 2. 掌握度评分体系
- **Anki 式**：4 级离散评分（Again/Hard/Good/Easy）
- **Duolingo 式**：连续概率值（0-100% 遗忘风险）
- **建议**：初期用 Anki 式 4 级评分，积累数据后升级为概率模型

### 3. 三阶段记忆管理
```
学习阶段 → 短间隔密集复习（分钟/小时级别）
    ↓
指数阶段 → 逐步拉长间隔（天/周/月级别）
    ↓（遗忘时）
重新学习 → 回退到学习阶段，但使用之前的指数间隔 × 遗忘乘数
```
- **关键参数**：遗忘乘数默认 0.1（新间隔 = 旧间隔 × 0.1）

### 4. 延迟复习奖励机制
- 若卡片在最佳复习日期后 n 天才被复习且仍答对
- 间隔奖励 = n / 难度分隔值
- 防止算法低估用户的实际掌握程度

## 5.2 内容组织层面

### 5. 知识图谱 + 前置依赖
- **Khan Academy 模式**：知识节点 → 前置依赖 → 解锁路径
- **适用场景**：学科类学习（法考、数学、编程等）
- **数据结构**：
```json
{
  "nodeId": "civil_law_contract",
  "title": "合同法",
  "prerequisites": ["civil_law_basic", "civil_law_legal_act"],
  "masteryRequired": 0.8,
  "children": ["civil_law_breach", "civil_law_remedy"]
}
```

### 6. 知识点标签 + 弱点追踪
- 每道题绑定知识点标签（如"竞合""共犯""代理"等）
- 错题自动按知识点聚类
- 生成弱点热力图

### 7. 多维度内容分组
- **按科目**：一级分类
- **按知识点**：二级分类
- **按难度**：三级标签
- 支持多条件筛选出题

## 5.3 个性化层面

### 8. 动态出题优先级
- **未做题优先**：优先推送未接触过的内容
- **错题插队**：答错的题目在合适的时间重新出现
- **稳固题降频**：多次答对的题目大幅拉长间隔

### 9. 会话级 vs 全局级数据分离
- **会话级**：本次刷题的错题、用时、正确率 → 生成即时反馈
- **全局级**：长期统计、遗忘曲线、科目趋势 → 生成长期建议

### 10. 掌握度可视化
- 进度条 + 百分比 → 直观
- 科目热度图 → 一目了然
- 正确率趋势折线图 → 追踪进步

# 六、总结

| 需求场景 | 推荐借鉴方案 |
|----------|-------------|
| 需要最简实现 | SM-2 算法（3 个参数，50 行代码） |
| 需要精准记忆 | SM-2 + 延迟奖励 + 随机噪声 |
| 需要理解式学习 | 知识图谱 + 前置依赖 |
| 需要个性化 | 累积数据后过渡到 ML 回归模型 |
| 需要游戏化 | Quizlet 式连对机制 + Duolingo 式进度显示 |
| 近期可落地 | SM-2 评分体系 + 错题知识点聚类 + 未做题优先排序 |

**关键洞察**：所有高效记忆系统的核心思想高度一致——"在最容易遗忘的时刻，以最小的复习量，达成最大的记忆强化"。区别只在于如何精确找到这个"时刻"：SM-2 用经验公式，HLR 用机器学习，但两者的哲学内核完全相同。
