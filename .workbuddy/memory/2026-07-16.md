## 第二批修复（已上线 commit 814878a）——“做过的题仍显示新题”
根因：新题标签仅看本地 `d.mem[题号]` 是否存在。做过却仍是新题 = mem 丢失/未写入，三个来源已全部修复：
1. **跨设备 mem 未合并（主因）**：`pullGist` 合并门槛过窄（仅看 t/w/todayLog/examScores），本地 t 不小于远端时不合并→仅在另一设备做过的题 mem 不进本地。改为把 mem/done/notes/flags/firstSeen 记录数也纳入触发条件（合并为非破坏性并集，门槛宜宽）。
2. **清空错题误删记忆**：`clearErrors` 原本 `delete d.mem[id]`，退回新题。改为只清 `d.w`，保留 mem，并加 debounceSync。
3. **组卷模式中途退出丢进度**：paper 模式 mem 仅在 endExam 交卷时写。新增 `EXAM_COMMITTED` 集合 + `commitExamQ(q)`，在 nextQ/skipQ/prevQ 离开已作答题时实时回流（firstSeen/done/错题/mem/kw），每题仅回流一次；endExam 循环跳过已回流题避免 SM-2 重复推进，并补齐 firstSeen 与正确题的 done。

## 第三批修改（已上线 commit c7eaaba）——AI知识点稳定 + 首页来源进度分列
1. **AI 知识点每次刷到都变**：`callDeepSeek` 每次提交答案都重新抽 topic 并**无条件覆盖** `d.topicFixes[题号]`；而防重复缓存 `aiCache` 是内存变量，刷新/重开即清空→再次请求 DeepSeek(temp=0.5 非确定性)抽出新词覆盖库值。
   - 修复：加“只生成一次”保护——仅当 `topicFixes[题号]` 为空才写入并迁移 kw；已存在则复用库值、绝不覆盖。写库+Gist 同步逻辑保留，跨设备/刷新一致。（按用户要求：不加“重新生成”按钮）
2. **首页自由刷题卡片进度分列**：卡片新增 `free-prog` 两列，显示「已做真题 X/真题总数」「已做模拟 X/模拟总数」+ 进度条。
   - 分母(真题/模拟总数)与分子(已做数)均来自题库 Q 按 source 统计；`updateHomeProgress` 中 Q 载入后遍历计算。
   - 后台静默预载题库（`if(!D) loadData(...)` 在 init 触发）以在未刷题时也填充分母；未载入时显示“加载中”。

## 第四批修改（已上线 commit fb8a7e5）——AI知识点不限字数 + 同步不覆盖题库说明
1. **AI 知识点去字数限制**：用户要求 topic 不限制字数、只要求简洁。
   - 提示词第1476行去掉“（不超过8字）”，改为“用最简洁的语言概括本题核心考点（力求简短，几个字到十几个字即可，不要啰嗦）”。
   - 写入块第1493-1494行去掉 `topicFix.length<=16` 守卫与 `substring(0,8)` 截断，仅保留 `topicFix.split('.')[0].trim()`（取首句并去空格）。“只生成一次、已存在不覆盖”逻辑不变。
2. **同步不会重置 topic（已向用户解释）**：
   - 题库文件 all_questions.json.gz 在 App 中是**纯只读**（doFetch 仅拉取、idbPut 仅本地缓存、Q=d 读内存），App 无任何写回服务器的逻辑；只有我改数据时用 git push 才更新。
   - AI topic 存在个人进度 `d.topicFixes[题号]`（localStorage），经 Gist pullGist/cloudSyncPush 在设备间同步；同步只传进度对象 d，不含题库，故不会覆盖网站题库或重置 topic。`topicFixes` 用 mergeObj 并集合并，已生成的不会被对端清空。
