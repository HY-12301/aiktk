# 精准记忆出题算法设计方案

## 一、设计目标

为法考刷题系统设计一套基于**间隔重复（Spaced Repetition）**和**动态掌握度评估**的出题算法，实现：
- 新题优先发现薄弱点
- 错题在遗忘临界点精准重现
- 已掌握题目以指数递减频率复习
- 每道题有独立的记忆档案

---

## 二、核心数据结构：题目记忆档案

每道做过的题目维护一个 `MemState` 对象：

```javascript
{
  ef: 2.5,           // 易度因子（Easiness Factor），初始 2.5
  interval: 0,       // 当前间隔（天）
  reps: 0,           // 连续答对次数
  nextReview: null,  // 下次复习日期（ISO 字符串）
  lastReview: null,  // 上次复习日期
  totalAttempts: 0,  // 总尝试次数
  totalWrong: 0,     // 总答错次数
  phase: 'new',      // 'new' | 'learning' | 'exponential' | 'relearning'
  learningSteps: [   // 学习阶段的固定步骤（分钟）
    {interval: 30, done: false},
    {interval: 120, done: false},
    {interval: 1440, done: false}  // 1天 = 1440分钟
  ],
  learningStepIdx: 0,
  reLearningStepIdx: 0,
  stability: 0      // 记忆稳定性（越高越不容易忘）
}
```

存储位置：`GS().mem`（localStorage 中的独立字段，与 done 并行）。

## 三、出题优先级排序

每次开始一轮刷题，按以下优先级从匹配条件的题目池中选取：

### 优先级 1：到期复习（距上次复习超过 nextReview）

```
逾期天数 = 今天 - nextReview 日期
逾期越久 → 越优先出现（遗忘风险最高）
如果逾期但用户仍然答对 → 给予"延迟奖励"（间隔 × 奖励系数）
```

### 优先级 2：学习阶段中的题目

```
处于 30min / 2h / 1d 固定步骤中的卡片
步骤到期 → 插入队列
```

### 优先级 3：错题高频知识点

```
同一个知识点下错题越多 → 该知识点下所有题目的优先级提升
（知识点交叉强化：做对一道题可能同时强化同知识点的其他题）
```

### 优先级 4：未做过的新题（随机打散）

```
全新题目 → 随机排列，避免同一天创建的题目始终在一起
```

### 优先级 5：已掌握题目（间隔最大者优先）

```
EF > 2.3 且 nextReview 在未来的题目
间隔越大 → 越久没复习 → 优先于间隔小的题目
```

## 四、评分与记忆状态更新

用户提交答案后，根据作答结果更新记忆状态：

### 4.1 质量评分（q，0-5 分）

| 场景 | q 值 | 描述 |
|------|------|------|
| 单选且答错 | 0 | 完全不会 |
| 多选且全错 | 1 | 方向性错误 |
| 多选且部分对 | 2 | 有点印象但不全 |
| 答对但犹豫/用时久 | 3 | 努力回忆起来 |
| 答对且较顺畅 | 4 | 掌握良好 |
| 秒答且连对多次 | 5 | 已内化 |

### 4.2 EF 更新公式（SM-2）

```
若 q ≥ 3：
  EF' = EF + (0.1 - (5-q) × (0.08 + (5-q) × 0.02))
  下限：EF ≥ 1.3

若 q < 3：
  EF' = max(EF - 0.2, 1.3)
  状态降级为 relearning
```

### 4.3 间隔计算

```
若 phase == 'new' 或 phase == 'learning'：
  → 通过固定步骤推进（30m → 2h → 1d）
  → 全部步骤通过后进入 exponential 阶段

若 phase == 'exponential'：
  若 q ≥ 3：
    若 reps == 0: interval = 1（天）
    若 reps == 1: interval = 6（天）
    否则: interval = round(interval × EF)
    reps += 1
  若 q < 3：
    phase = 'relearning'
    learningStepIdx = 0
    reps = 0

若 phase == 'relearning'：
  → 重新走学习步骤，但最终间隔使用之前的 interval × 0.1
  → 答对全部步骤后回到 exponential 阶段

nextReview = today + interval
```

### 4.4 延迟奖励

```
若 q ≥ 3 且 实际复习日期 > nextReview：
  bonus = floor(延迟天数 / 难度分隔值)
  难度分隔值：
    q == 3（努力回忆）：4
    q == 4（良好）：2
    q == 5（轻松）：1
  interval = interval + bonus
```

### 4.5 已掌握判定

```
若 EF ≥ 2.5 且 interval ≥ 90（天）且 连续答对 ≥ 5 次：
  标记为"已掌握"，不再主动出题（除非手动选择复习）
```

## 五、会话级出题流程

```javascript
function buildQuizPool(subjects, sources, questionCount) {
  // Step 1: 获取匹配条件的全部题目
  let candidates = Q.filter(bySubjectAndSource);

  // Step 2: 按记忆状态分类
  let overdue = [];      // 到期未复习
  let learning = [];     // 学习阶段中
  let newQuestions = []; // 从未做过
  let reLearning = [];   // 重新学习中
  let mastered = [];     // 已掌握

  candidates.forEach(q => {
    let mem = getMem(q.id);
    if (!mem) { newQuestions.push(q); return; }

    if (mem.phase === 'mastered') { mastered.push(q); return; }

    let now = Date.now();
    let due = mem.nextReview ? new Date(mem.nextReview).getTime() : 0;

    if (mem.phase === 'learning' && due <= now) learning.push(q);
    else if (mem.phase === 'exponential' && due <= now) overdue.push(q);
    else if (mem.phase === 'relearning' && due <= now) reLearning.push(q);
    else {} // Not due yet, skip for now
  });

  // Step 3: 按优先级排序
  // P0: 到期复习（逾期天数 DESC）
  overdue.sort((a, b) => overdueDays(b) - overdueDays(a));

  // P1: 重新学习（逾期天数 DESC）
  reLearning.sort((a, b) => overdueDays(b) - overdueDays(a));

  // P2: 学习阶段（步骤序号 DESC，越接近完成越优先）
  learning.sort((a, b) => getMem(b.id).learningStepIdx - getMem(a.id).learningStepIdx);

  // P3: 新题（随机打散）
  shuffle(newQuestions);

  // P4: 已掌握（间隔最大者优先，偶尔抽查）
  mastered.sort((a, b) => getMem(b.id).interval - getMem(a.id).interval);

  // Step 4: 合并，取前 questionCount 题
  let combined = [...overdue, ...reLearning, ...learning, ...newQuestions];
  // 知识点交叉混排：同一知识点的题不超过连续 3 题
  combined = interleaveByKnowledgePoint(combined);

  return combined.slice(0, questionCount);
}
```

## 六、与现有系统的集成

### 6.1 存储

```javascript
function getMem(qid) {
  return (GS().mem || {})[String(qid)] || null;
}

function setMem(qid, state) {
  var d = GS();
  d.mem = d.mem || {};
  d.mem[String(qid)] = state;
  SS(d);
}
```

### 6.2 评分映射

现有系统中，A[q.id].a 是用户选择的答案，q.correctAnswer 是正确答案。

```
单选：A === correct → q=4（正确）/ q=0（错误）
多选：A === correct → q=4 / 部分对 → q=2 / 全错 → q=1
不定项：同多选
若连对次数 ≥ 3 且秒答 → q=5
```

### 6.3 兼容现有 done 数据

```
done 字典继续作为"是否做过"的标志
mem 字典新增为"记忆状态"的存储
两者独立维护，互不冲突
```

## 七、算法优势总结

| 特性 | 现有算法 | 新算法 |
|------|---------|--------|
| 新题优先 | ✓ | ✓ |
| 未做题优先 | ✓ | ✓（更高优先级排序） |
| 错题复现 | ✗（只存全局错题本） | ✓（按遗忘曲线计算最佳复现时间） |
| 已掌握降频 | ✗ | ✓（interval → 90天后暂停） |
| 延迟奖励 | ✗ | ✓（逾期答对 → 间隔加速） |
| 知识点交叉 | ✗ | ✓（同知识点题不连续 3 题以上） |
| 三阶段管理 | ✗ | ✓（learning → exponential → relearning） |
| 每道题独立记忆档案 | ✗ | ✓（EF/interval/reps/stability） |
